方案 A：单 Astro 项目，双域名部署

推荐方案。一个仓库，一个 Pages 项目，通过 Cloudflare 域名规则将根路径映射到不同域名。

项目结构
a-zan-site/
├── src/
│   ├── pages/
│   │   ├── index.astro          # a-zan.xyz 首页
│   │   ├── blog/
│   │   │   ├── index.astro      # blog.a-zan.xyz 列表
│   │   │   ├── [...slug].astro  # 文章详情
│   │   │   └── tags/
│   │   │       └── [tag].astro  # 标签页
│   │   ├── 404.astro
│   │   └── rss.xml.js
│   ├── content/
│   │   └── blog/                # Markdown/MDX 文章
│   ├── components/
│   ├── layouts/
│   └── config.ts                # 站点常量
├── public/
├── astro.config.mjs
└── package.json
Cloudflare Pages 域名配置
Pages 项目 a-zan-site
自定义域名 a-zan.xyz, blog.a-zan.xyz
Build command npm run build
Output directory dist
路由规则 无需 Workers，Pages 自动按请求 Host 服务同一路径

优点

  • 一个仓库，一套依赖，维护最简单
  • 主页和博客共享 layout、组件、样式变量
  • 一次 build/deploy，原子化上线
  • 未来新增 *.a-zan.xyz 工具站可复用组件
  • SEO 无跨域问题，sitemap/RSS 统一生成

缺点

  • 两个站点耦合在同一版本线
  • 博客构建量大会拖慢主页 trivial 修改
  • 权限粒度粗（但个人项目无影响）
